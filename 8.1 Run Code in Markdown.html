在 Markdown 中运行代码

运行 Markdown 中代码，有何意义？

如果只是运行代码，是没有多少意义的。真正有意义的是代码本身，它是重复劳动高度提炼后自动化。
 MarkEditor 作为文本工具，工作目录结合文件树的基础结构，有非常棒的笔记、备忘的使用体验。而不被使用的笔记，没有多大的意义，所以，我们想让这些笔记用起来。
 在 MarkEditor 中的代码块中增加可运行的机制之后，一下子打开了个人工作脚本自动化的新世界。
 比如，作为设计者本身，我们不是用这个功能来写 hello world 的，而是直接管理一些服务的部署、Docker 容器的维护……

如何使用？

将光标点中代码(高亮)的片段，然后右键点击，选择“运行代码”即可。而右键有没有对应的命令显示，具体视本地的开发环境是否已搭建，比如 python 就意味着要有 python 可直接调用，ruby 则是 ruby 必须已安装，而 javascript 则依赖于node .etc 

# 如本文顶部内容的 meta 信息，若设置了 dir 目录，可以直接 import 对应路径下的 package 进行调用
print 'hello world'
for i in range(3):
    print i

// 实际需要 nodejs 的环境支持，调用的是 node 这个命令
console.log('run in nodejs in fact')
var a = 1;
var b = 2;
console.log(a+b)

注意:
 1， MarkEditor 基础构架的问题，不支持 python3 的运行。
 2，如果运行的是 python，默认会在脚本前添加一定的 sys.path，以保证运行，这部分内容，你也可以自定义(Windows 一般不用处理)。Command+Shift+C 打开配置目录，添加 python.py，里面的内容即是，Mac 上可以简单运行类似的命令行，一行搞定: python -c 'import sys; print "import sys; sys.path = (sys.path[:1] if sys.path else []) + "+str(sys.path[1:])' > ~/.MarkEditorData/python.py。
 3, 由于各个终端设备(Mac or PC)的开发环境迥异，不一定能保证代码块运行的机制能工作正常。
