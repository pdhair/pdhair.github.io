Customize Interface

自定义界面风格

可以在设置中选择日间，或者夜间模式进行定义。具体的定义项的说明，可以查看菜单栏 (Windows版本位于托盘按钮上) 自定义的帮助。
 MarkEditor几乎所有跟色彩有关的界面，都已开放出来自定义。如果自己看到朋友已经定义好的样式，感觉欢喜，可以让对方直接把定义的文本内容给你，粘贴到设置项中并应用即可。

跟色彩没有直接关系的一些地方:

  • 文件管理器: 它有三中表现形式，在对应的文件夹设置项中可以定义；
  • 底部栏: 可以通过**Command+0**这个快捷键切换是否显示，设置中也可以设定默认是否显示；
  • Tab栏: 但有且仅有一个Tab的时候，Tab栏会自动消失。

HTML预览的自定义

当你预览makdown最终内容，或者选择导出的时候，你可能想重新定义HTML的模板或者css样式。
 一般有两种方式:

直接修改MarkEditor的日间、夜间风格

这些界面相关的定义项，默认会最终生成对应的css文件，比如文字的颜色， 在正文中什么样子，在预览中也会什么样子。

重新定义HTML的渲染模板

在MarkEditor的设置里，你也可以自定义一个模板来渲染最终的HTML，还可以增加自己自定义CSS样式文件。

模板中需要指定%(content)s这个变量，这是Markdown转义过来原始的HTML内容，另外%(title)s一般放在<head><title>上，%(css)s这个变量对应的是一个CSS样式文件，它是由当前界面风格自动生成的，你也可以用自己的样式文件替代。<--header_scripts-->最终会被替换为流程图支持的js文件，<!--mathjax-->则是对应数学公式支持的js文件，如非自行替换，一般建议保留。

MarkEditor系统默认的模板代码如下: 

<!DOCTYPE html>
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en-us">
<head>
    <meta http-equiv="content-type" content="text/html; charset=utf-8" />
    <title>%(title)s</title>
    <link rel="stylesheet" href="%(css)s">
    <!--header_scripts-->
</head>
<body>
<div class="post">
    <div class="post_body">
        %(content)s
    </div>
</div>
<!--mathjax-->
</body>
</html>

自定义CSS样式

一般情况下，完整得自定义CSS样式，最好的办法是直接重新自定义HTML模板；但有时候，我们需要对MarkEditor的默认CSS文件进行微调，那么可以在设置窗口中，找到MD 模板中css自定义的地方，输入需要的css样式即可。
 但需要注意，MarkEditor 解析生成的 HTML 规则跟普通的 Markdown 解析结果是不一样的，ME 更倾向于 HTML DOM 结构的可扩展性。
 P 元素会自带一个 class md_block，其内部每一行都由 span 包裹，class 为 md_line。
 由<hr> 包裹的部分，会外部包一层 DIV，class 为 hr_part。
